문제해결 시나리오
에러가 하나 나온다 하이퍼바이저가 꺼져있을때 나오는 에러
에러가 직접 뜬 상황이 아니기때문에 윈도우 텐에서 에러구문이 나온다. 하이퍼 바이저가 꺼져있다고
윈도우 프로그램 추가/제거에서 (앱 및 기능에 프로그램 및 기능에 가서 보면 윈도우 기능 켜기/끄기에 체크 해주어야한다)
도커에서 할때는 꺼져 있어도 되기는 하는데 저거때문에 에러가 나면 찾아서 체크해 주는게 해결책
에러가 발생하는데 풀을 땡길때 에러 내용을 보면 권한이 없다. 연결이 안되어서 유저 네임이랑 패스워드를 넣어라
로그인이 안되어 있어서 나오는 에러 도커허브에 로그인을 하면 된다.

서술형
1. 호스트 가상화가 뭔지 서술하여라 (가상환경 운영체제 기존에 운영체제 그 위에 가상 소프트웨어를 운영하는게 호스트 가상화 (게스트))
호스트 os에 가상화 sw를 설치해서 이 가상화 sw에서 Guest os를 작동 시키는 기술

2. 이미지 목록을 출력할 때 사용하는 명령어가 뭔지
docker images

3. 도커 특징 중 상호 운영성이 뭔지 (시스템에 연계해서 사용할 수 있다.)
여러 조직이나 시스템과 연계하여 사용할 수 있는 소프트웨어의 특성

4. 도커 push 명령어가 나온다. 명령어가 의미하는 바가 무엇인지 써라 (명령 문장이 있는데 그 문장을 풀어서 기술)
계정이랑 이미지랑 슬래쉬 앞에 보통 계정명이 들어가는데 그 내용이 나와있음 그거를 풀어서 서술
 docker push peksoon/iksoon_tomcat:1.0.2


5. 도커허브가 뭔지 기술
도커에서 운영하는 도커 이미지 저장소 서비스, 도커허브를 통해 이미지를 공유하거나 다운받을 수 있다.

6. 도커허브에 레이어방식으로 이미지가 생성이 되어 있는데 다운을 받으면 이미지 레이어가 하나씩 다운받아진다.
도커허브에서 이미지 여러개를 처리하는 방식에 대해서 서술해라 (베이스 이미지로 톰캣 버전을 잡고 수정을 해서 올리면
수정한거를 레파지토리에 올렸을때 해당하는 파일들이 전부 저장이 되지 않는다. 기존에 도커 베이스이미지 같은 경우는
공식적으로 제공되고 있는것이므로 걔는 마운트된다. (일단 불러와지고 수정한 레이어만큼 수정이 되서 레이어 방식으로 처리된다.)

기존 이미지에 파일 하나가 추가되었다고 처음부터 전부 다 다운받는 비효율적인 방식이 아니라 
추가적인 파일이 필요한 경우에 해당 파일만 추가적으로 다운받는 방법, 
이미지는 여러 개의 읽기 전용 레이어로 구성되고 파일이 추가되거나
수정되면 새로운 레이어가 생성된다.
